## How it was fixed (2026-07-16)

1. **`label` (285)** — `settings_components/Members.js`: per-row `aria-label` on all six
   member checkboxes ("Web working group — {name}", …).
2. **`select-name` (5)** — `aria-label` on `DropdownSelect` (status level), the plans
   sort `<Select>` (`PlansList.jsx`), and the shared `PersonAssignmentSelector` /
   `EntityAttachmentSelector` selects (placeholder text is not an accessible name).
3. **`color-contrast` (536)** — token-level:
   - App-wide sweep: text `gray.400`/`gray.500` → `gray.600` (107 files, 374 subs);
     `gray.600` is now the documented AA floor for text grays (design-sense §2).
   - `theme.js` component overrides: `Stat` label/helpText → gray.600 + opacity 1;
     `Badge`/`Button` solid re-shaded per colorScheme (`SOLID_AA_SHADE`); Badge outline
     + Button outline/ghost text re-shaded for the light schemes.
   - `statusColors.js`: new `getStatusTextColor()` (AA-dark ramp). White-on-ramp status
     badges converted to tint bg + dark text (report tables, evidence quality panel,
     status level details, email export). `StatusLevelLadder` labels and `StatusPill`
     text use the text ramp. Removed the duplicate buggy `getStatusColor` in `tools.js`.
4. **`aria-prohibited-attr` (19)** — `DescriptorHelp.js` `HelpTip`: `role="note"` on the
   labelled tooltip trigger span.
5. **`nested-interactive` (10)** — `IndicatorRow.jsx`: container `role="button"` removed;
   disclosure semantics (`aria-expanded`, named `aria-label`) moved onto the real
   chevron button (APG Disclosure).
6. **`scrollable-region-focusable` (3)** — `PeopleList`/`PrincipleList` scroll boxes and
   the About `VocabTable` `TableContainer`: `tabIndex={0}` + `role="region"` +
   `aria-label` + visible focus ring.

Keyboard-contract work (roving tabindex, skip link, focus-on-navigate, one section-switcher
idiom) is deliberately NOT here — that's the second pass:
`claude_files/a11y-apg-second-pass-plan.md`.
